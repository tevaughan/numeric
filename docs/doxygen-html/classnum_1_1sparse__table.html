Detailed Description

template<typename A, typename F>
class num::sparse_table< A, F >

A piecewise function that, in logarithmic time, looks up the sub-function appropriate to the argument.

For any argument \( a \), the corresponding sub-function and its sub-domain are found in logarithmic time. The sub-domains are contiguous and of variable length. If \( a \) be less than the least element of the first subdomain or greater than the greatest element of the last subdomain, then there is no corresponding sub-function, and zero is returned.

Stored in a table of \( n \) records, the sub-function corresponding to \( a \) is \( f_i \), and the center of the corresponding subdomain is \( a_i \), where \( i \in \{0, 1, \ldots, n-1\} \). The sub-argument \( a - a_i \) is passed to \( f_i \), and the value returned by \( f_i \) is returned from the lookup.

In the simplest, fastest table, each \( f_i \) ignores its argument and returns a constant value. However, the infrastructure provided by sparse_table allows for logarithmic-time lookup of any kind of sub-function (for example, a cubic interpolant).

To implement logarithmic-time lookup, an instance of sparse_table stores \( n \) triplets \( (a_0, \Delta a_0, f_0) \), \( (a_1, \Delta a_1, f_1) \), \( \ldots \), \( (a_{n-1}, \Delta a_{n-1}, f_{n-1}) \), sorted so that \( a_0 < a_1 < \cdots < a_{n-1} \), where \( \Delta a_i \) is the length of the sub-domain whose center is \( a_i \).

The lookup occurs by way of operator()(). When \( a > a_0 - \frac{\Delta a_0}{2} \), and \( a < a_{n-1} + \frac{\Delta a_{n-1}}{2} \), record \( i \) is found so that \( |a - a_i| \leq \frac{{\Delta a}_i}{2} \).

operator()() returns \( f_i(a - a_i) \).

See dense_table for a type that usually can approximate a given function so precisely as sparse_table but with faster lookup. The cost is that dense_table requires more storage for the same precision with the same sub-function type F.
