2 # Introduction {#mainpage}
3 
4 The C++-11 numeric library provides physical units, an adaptive quadrature
5 integrator, and the ability to instantiate a linear interpolant in various,
6 convenient ways.
7 
8 ## Physically dimensioned quantities.
9 
10 The implementation of physical units with compile-time checking for dimensions
11 allows one to do things like this:
12 
13 ```cpp
14 #include <iostream>
15 #include <units.hpp>
16 
17 using namespace num;
18 using namespace std;
19 
20 int main()
21 {
22  // 'ft' stands for one foot.
23  length ell = 2.1 * ft; // length of pendulum
24  // 'gn' stands for one standard gravity.
25  frequency f = sqrt(gn / ell); // frequency of small oscillations
26  cout << f << endl; // [3.9142 s^-1]
27  return 0;
28 }
29 ```
30 
31 The compiler would produce an error if, say, a distance were assigned to a
32 speed.

36 ## Integration
37 
38 An adaptive-quadrature, trapezoid-rule, global integration function,
39 num::integral allows the definite integral of any continuous function of one
40 variable to be computed a specified tolerance.
41 
42 The global num::integral is designed to be able to interoperate with units.
43 
44 However, if dimensioned quantities be not used, then one need only include
45 [integral.hpp](\ref integral.hpp) and need not link against `libnumeric.so`.
46 
47 [Details of integration.](\ref integrals)

49 ## Linear Interpolation
50 
51 There are two basic ways to construct a linear interpolant:
52 
53 1. from a list of points and
54 2. from a continuous function and an interval on its domain.
55 
56 Once a linear interpolant is constructed, it can be integrated efficiently and
57 exactly over any interval. An interpolant can also be multiplied by another
58 interpolant or by a scalar without any loss of information.
59 
60 When the interpolant is constructed from a continuous function, the same
61 algorithm employed in num::integral is used, but the intermediate results of
62 the adaptive subdivision are stored for use as points for interpolation. So
63 the interpolant is intelligently constructed to have a high density of points
64 only where necessary to ensure an accurate representation of the function and
65 its integral.
66 
67 The num::interpolant class is designed to interoperate with units.
68 
69 However, if dimensioned quantities be not used, then one need only include
70 [interpolant.hpp](\ref interpolant.hpp) and need not link against
71 `libnumeric.so`.
72
