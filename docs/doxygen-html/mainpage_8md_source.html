2 # Introduction {#mainpage}
3 
4 The numeric library provides
5 
6  - header-only classes for numerical integration and for interpolation and
7 
8  - a compiled library (linked with '-lnumeric') for dimensioned quantities,
9  which can be used with the integrator and with the interpolator.

11 ## Installation
12 
13 Grab the latest 'tar.gz' file from [the releases
14 page](https://github.com/tevaughan/numeric/releases).
15 
16 Or clone from [github](https://github.com/tevaughan/numeric).

19 ```
20 tar fvxz numeric-x.y.tar.gz
21 cd numeric-x.y
22 ./configure
23 make
24 sudo make install

29 ## Physically dimensioned quantities.
30 
31 Base class 'dimval' has public default constructor for zero-valued quantity.
32 
33 ~~~{.c++}
34 #include <units.hpp>
35 dimval<-1,1,0,0,0> foo; // A speed with initial value of zero.
36 // another comment
37 ~~~
38 
39 Each of dimval's template value parameters is an integer representing the
40 exponent of one of five fundamental units.
41 
42 1. second
43 2. meter
44 3. kilogram
45 4. coulomb
46 5. kelvin
47 
48 Aliases are available for convenience.
49 ```.c++
50 #include <units.hpp>
51 num::speed foo; // A speed with initial value of zero.
52 ```
53 
54 A wide array of units can be used for initialization and computation.
55 ```c++
56 #include <units.hpp>
57 num::speed v = 2.3 * mi / hr; // Speed with initial value 2.3 miles/hour.
58 ```
59 
60 sqrt() and fabs() are supported.
61 
62 Member functions allow for integer power or root.
63 ```c++
64 #include <units.hpp>
65 using namespace num;
66 length x = 2 * cm;
67 area a = x.pow<2>();
68 length y = 0.5 * sqrt(a);
69 volume v = x.pow<3>();
70 length z = 0.1 * v.root<3>();
71 ```

73 ## Linear interpolation
74 
75 A linear interpolant can be initialized directly from a file with two
76 space-delimited columns. Then it can be scaled, multiplied by another
77 interpolant with no loss of information, integrated, etc.
78 
79 ```c++
80 #include <iostream>
81 #include <interpolant.hpp>
82 #include <units.hpp>
83 
84 using namespace num;
85 using namespace std;
86 
87 // Read in first two columns of space-delimited file.
88 // First column is interpreted as a double-precision length in nanometers.
89 // Second column is interpreted as a double-precision time in seconds.
90 // Explicit namespace scoping for num::time might be required to avoid
91 // conflict with C library's time() function.
92 interpolant<length, num::time> i("file.txt", nm, s);
93 
94 // Read in first two columns of space-delimited file.
95 // Each column is interpreted as a double.
96 interpolantd j("file.txt");
97 
98 int main()
99 {
100  cout << i(500 * nm) << endl;
101  cout << j(500) << endl;
102  // Class interpolant knows how to integrate itself exactly.
103  cout << i.integral(300 * nm, 900 * nm) << endl;
104 
105  // Product of interpolants is interpolant with no loss of information.
106  interpolant<length, double> k("file2.txt", nm, 1.0);
107  interpolant<length, num::time> m = i * k;
108 
109  return 0;
110 }
111 ```
112 
113 A linear interpolant can also be initialized from an existing, continuous
114 function.
115 
116 ```.c++
117 #include <cmath> // for erf()
118 #include <interpolant.hpp>
119 
120 using namespace num;
121 
122 int main()
123 {
124  // 'interpolantd' is an alias for 'interpolant<double, double>'.
125  interpolantd const i(erf, -1.0, 2.0);
126 
127  // Now you can use, for example i(1.3), to do a quick table-lookup of erf(),
128  // etc. This is useful especially for a continuous function that takes a
129  // long time to calculate, when it has to be used over and over again in a
130  // loop.
131 
132  return 0;
133 }
134 ```

136 ## Integration
137 
138 ```c++
139 #include <iostream>
140 #include <integral.hpp>
141 #include <units.hpp>
142 
143 using namespace num;
144 using namespace std;
145 
146 area square(length x) { return x * x; }
147 
148 int main()
149 {
150  volume const i = integral(square, 0 * cm, 1 * cm);
151  cout << i << endl; // "[3.33333e-07 m^3]"
152  return 0;
153 }
154 ```
