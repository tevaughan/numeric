2 # Physically Dimensioned Quantities {#dimvals}
3 
4 Class num::dimval is a model of a dimensioned quantity. In the physical
5 sciences, every product of a number and a physical unit is a dimensioned
6 quantity.
7 
8 A physical unit is not a number but can be multiplied by a number or by another
9 unit. For example, if a ruler be twelve inches long, then one might write the
10 length of the ruler as \f$12~\text{in}\f$. In this case, \f$\text{in}\f$
11 refers to a physical unit, an inch. In the example, \f$12~\text{in}\f$
12 indicates that the number 12 is multiplied by an inch. In ordinary language,
13 one has that twelve times an inch is twelve inches. In this example, the
14 quantity under consideration is a length, which is one kind of dimensioned
15 quantity. The same 12-inch length could be expressed as \f$30.48~\text{cm}\f$.
16 So different units can correspond to the same dimension, which in the present
17 example is the dimension of length.
18 
19 Every instance of num::dimval corresponds to one or another dimension, such as
20 length, speed, mass, force, etc. The dimension is specified by the template
21 value parameters of num::dimval, which is a template class. Each of dimval's
22 template value parameters is an integer representing the exponent of one of
23 five fundamental units.
24 
25 1. second
26 2. meter
27 3. kilogram
28 4. coulomb
29 5. kelvin
30 
31 Class num::dimval has public default constructor for a zero-valued quantity.
32 ```cpp
33 #include <units.hpp>
34 dimval<-1,1,0,0,0> foo; // A speed with initial value of zero.
35 ```
36 
37 For many a dimension, an alias is available for convenience.
38 ```cpp
39 #include <units.hpp>
40 num::speed foo; // A speed with initial value of zero.
41 ```
42 
43 The definition of the aliased dimensions is stored in a text file
44 [dimensions.txt](https://github.com/tevaughan/numeric/blob/master/src/dimensions.txt)
45 that is used at library build time to generate the corresponding C++ file
46 dimensions.hpp. This file can easily be edited before the library is built to
47 increase the number of known dimensions.
48 
49 For initialization of a dimensioned quantity, a wide array of units can be used
50 for initialization and computation. The definition for every unit known by the
51 library is stored in a text file
52 [units.txt](https://github.com/tevaughan/numeric/blob/master/src/dimensions.txt)
53 that is used at library build time to generate the corresponding C++ file
54 units.hpp.
55 ```cpp
56 #include <units.hpp>
57 using namespace num;
58 speed v = 2.3 * u::mi / u::hr; // Speed with initial value 2.3 miles/hour.
59 ```
60 
61 sqrt() and fabs() are supported.
62 
63 A member function allows for an integer power, and another member function
64 allows for an integer root.
65 ```cpp
66 #include <units.hpp>
67 using namespace num;
68 length x = 2 * u::cm;
69 area a = pow<2>(x);
70 length y = 0.5 * sqrt(a);
71 volume v = pow<3>(x);
72 length z = 0.1 * root<3>(v);
73 ```
74 
75 Standard-library output streams are supported.
76 ```cpp
77  #include <iostream>
78  #include <units.hpp>
79  using namespace num;
80  int main() {
81  using time = num::time; // in case of conflict with C library's time()
82  length x0 = 2 * u::m; // meters
83  speed v = 5 * u::m / u::s; // meters per second
84  time t = 3 * u::min; // minutes
85  // Output distance in meters (the default) and in yards.
86  std::cout << x0 + v * t << std::endl; // "[902 m]"
87  std::cout << (x0 + v * t) / yd << " yd" << std::endl; // "986.439 yd"
88  return 0;
89  }
90 ```
