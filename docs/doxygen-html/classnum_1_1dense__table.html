Detailed Description

template<typename A, typename F>
class num::dense_table< A, F >

A piecewise function that, in constant time, looks up the sub-function appropriate to the argument.

For any argument \( a \), the corresponding sub-function and its sub-domain are found in constant time. The sub-domains are contiguous and of common length \( \Delta a \). If \( a \) be less than the least element of the first subdomain or greater than the greatest element of the last subdomain, then there is no corresponding sub-function, and zero is returned.

Stored in a table of \( n \) records, the sub-function corresponding to \( a \) is \( f_i \), and the center of the corresponding subdomain is \( a_i \), where \( i \in \{0, 1, \ldots, n-1\} \). The sub-argument \( a - a_i \) is passed to \( f_i \), and the value returned by \( f_i \) is returned from the lookup.

In the simplest, fastest table, each \( f_i \) ignores its argument and returns a constant value. However, the infrastructure provided by dense_table allows for constant-time lookup of any kind of sub-function (for example, a cubic interpolant).

To implement constant-time lookup, an instance of dense_table stores

  • \( a_0 \), the center of the first sub-domain,
  • the common difference \( \Delta a = a_{i+1} - a_i \) (for \( i \in \{0, 1, \ldots, n-2\} \)) between any center of a sub-domain and the next center, and
  • a list of the \( n \) sub-functions \( f_0 \), \( f_1 \), \( \ldots \), \( f_{n-1} \).

Thus a regular grid is established across the value of the argument.

The lookup occurs by way of operator()(). When \( a > a_0 - \frac{\Delta a}{2} \), and \( a < a_{n-1} + \frac{\Delta a}{2} \), the offset \( i \) is found as the integer truncation of \( \frac{a - a_0}{\Delta a} + 0.5 \).

operator()() returns \( f_i(a - a_i) \).

See sparse_table for a piecewise function that can approximate a function so precisely as dense_table but with fewer sub-functions of the same type F. The cost of a smaller table in sparse_table is logarithmic-time lookup.
